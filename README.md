# AR 盲盒开盒互动 8th Wall 版本

这是一个独立的新工程，用 8th Wall world tracking + Three.js 做真实摄像头场景里的盲盒开盒流程。

## 当前实现

- 点击屏幕中央的“获取盲盒”，取当前摄像头中心射线与虚拟地面的交点，把 3D 盒子放到这个真实场景位置。
- 盒子从屏幕上方自然落下，落地后停留在世界坐标中，手机移动时会产生空间透视变化。
- 盒子上方有 3D 撕拉条和虚拟卡通手，用户需要从左向右拖动撕开。
- 撕开后随机抽取 `assets/prizes/card-01.png` 到 `card-09.png` 中的一张圆角白色小卡。
- 合影页使用 `assets/prizes/character-01.png` 到 `character-09.png` 的透明人物图，不带白边和白色背景。
- 默认 3D 盒子已经贴上处理后的包装照片贴图：`assets/box-textures/front.jpg`、`back.jpg`、`left.jpg`、`right.jpg`、`top.jpg`、`bottom.jpg`。
- 小卡从盒子里弹出，落到盒子右侧并躺在桌面上。
- 点击真实场景里的小卡位置，小卡会被“捡起”，在屏幕中央闪亮旋转展示。
- 点击“与抽到的角色合影”进入合影页，可切换前后摄像头、拖动角色位置、合影定格、分享图片、进入购买链接。

## 测试方式

1. 打开 8th Wall Desktop。
2. 选择本文件夹：

   `C:\Users\21862\Documents\大广赛\blindbox-8thwall-openbox`

3. 8th Wall Desktop 正常运行后，桌面浏览器通常会显示二维码页。
4. 用手机浏览器扫描二维码打开，允许摄像头权限。
   如果用 ngrok 免费链接打开时出现 `Visit Site` 提示页，需要先点 `Visit Site` 进入。
5. 手机对准桌面中心，点击“获取盲盒”。
6. 移动手机观察盒子是否停在刚才的真实桌面位置，而不是贴在屏幕上。
7. 移到盒子上方，拖动虚拟手从左向右撕开拉条。
8. 等小卡落到盒子右侧后，点击小卡位置。
9. 小卡中央展示后，点击“与抽到的角色合影”，测试合影页功能。

## 验收重点

- 盒子不是固定在屏幕中心，而是放在点击时捕捉到的真实场景位置。
- 撕拉交互作用在 3D 盒子上的虚拟拉条，不是普通页面按钮。
- 小卡点击使用 3D 射线命中检测，需要点到落在场景里的卡片。
- 合影页仍然保持摄像头画面工作，角色可以拖动，合影按钮会把当前画面固定成图片。

## 后续替换素材

- 当前盒子是默认 3D 长方体模型，六个面已经使用你提供的商品照片做贴图。
- 如果之后有更清晰的六面扫描图，可以替换 `src/assets/box-textures` 里的同名贴图。
- 如果之后拿到正式 `.glb/.gltf` 模型，可以用 Three.js 的 `GLTFLoader` 替换默认盒子，并保留现有放置、撕拉、小卡、合影流程。
