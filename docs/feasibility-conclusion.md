# 移动端 WebAR 技术路线可行性结论

## 结论

当前项目已经证明：使用 MindAR + A-Frame + 静态网页部署的方式，可以实现泡泡玛特 Zsiga 在开学季传播节点下所需的基础 WebAR 互动体验。

这条路线不仅可行，而且已经完成到“基础正式版”阶段。

## 已验证能力

- 手机网页可以加载 AR 页面。
- 页面可以读取 MindAR 的 `targets.mind` 识别文件。
- 浏览器可以请求摄像头权限。
- 摄像头可以识别泡泡玛特官方竖版海报。
- 识别成功后可以叠加 AR 视觉内容。
- 页面按钮可以切换不同广告信息层。

## 当前适合提交的作品形态

当前推荐包装为：

```text
Zsiga 开学季 AR 情绪盲盒
```

它的核心体验是：

```text
扫码进入网页 → 对准官方海报 → 开启 AR 情绪盲盒 → 抽取情绪能量卡 → 截图分享或跳转泡泡抽盒机
```

## 优势

- 不需要安装 App。
- 用户扫码即可进入。
- 技术成本低，适合学生团队实现。
- 静态网页即可部署，后期方便放到 HTTPS 平台。
- 互动逻辑可以继续扩展为真实导流和传播闭环。

## 风险和优化方向

| 风险 | 处理方式 |
| --- | --- |
| 手机浏览器不允许摄像头 | 使用 HTTPS 部署 |
| 微信内置浏览器兼容性不稳定 | 提示使用 Safari / Chrome 打开 |
| 识别图细节不够导致识别慢 | 保持官方海报版本稳定，必要时重新生成 `targets.mind` |
| 素材太大导致加载慢 | 压缩图片资源，保持 AR 素材轻量 |
| AR 内容漂移 | 使用更稳定的识别图，保持光线充足 |

## 下一步

技术链路已经跑通，并且已经切入泡泡玛特官方海报识别。接下来不应该继续堆技术，而是完善参赛闭环：

- 继续细化四层 AR 卡片和真实导流动作。
- 录制 60-90 秒演示视频。
- 部署 HTTPS 页面并生成二维码。
- 补充 iPhone Safari / Android Chrome 真机测试记录。
